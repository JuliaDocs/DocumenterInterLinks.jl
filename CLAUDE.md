## Overview

`DocumenterInterLinks.jl` is a Julia plugin for [Documenter.jl](https://documenter.juliadocs.org/) that enables cross-project linking via `@extref` link targets. It uses inventory files (generated by Documenter or Sphinx) to resolve external references. The backend for reading inventory files is [DocInventories.jl](https://juliadocs.org/DocInventories.jl/stable/).

## Common Commands

The project uses a `Makefile` for development. The test environment is in `test/Project.toml` (not the root `Project.toml`); `test/Project.toml` uses a `[sources]` entry so `Pkg.instantiate()` resolves the local package without `Pkg.develop()`. Run `make help` to see the different tasks. These include:

```bash
make test        # Run full test suite
make coverage    # Run tests with coverage and print summary table
make htmlcoverage  # Run tests with coverage and generate HTML report in ./coverage
make docs        # Build documentation
make codestyle   # Apply JuliaFormatter to all source files
make distclean   # Restore to clean checkout state (removes all Manifest.toml files)
```

**From within the dev REPL** (`make devrepl`):

```julia
include("test/runtests.jl")          # Run full test suite in-process
include("test/test_interlinks.jl")   # Run a single test file
include("docs/make.jl")              # Build docs
```

## Architecture

The package exports two plugins:

### `InterLinks` (`src/interlinks.jl`)
The main plugin struct (`<: Documenter.Plugin`). Maps project name strings to `DocInventories.Inventory` instances. Constructed from key-value pairs where values can be a root URL, a URL to an inventory file, a tuple of (root URL, inventory sources…), or a `DocInventories.Inventory`. Implements ordered-dict iteration. The `find_in_interlinks(links, extref)` function is the core resolution function: given an `@extref` string, it returns the full URL.

### `ExternalFallbacks` (`src/fallback.jl`)
An optional secondary plugin (`<: Documenter.Plugin`). Maps Documenter `@ref` slugs to `@extref` strings. When `automatic=true`, it searches all inventories for unresolved `@ref` links. Implements the `ExternalFallbackResolver` pipeline step in `Documenter.XRefResolvers` (order 10.0).


## General Guidelines

* Make sure to only use explicit imports in Julia code, and that there are no imported functions or constants that are not actually used.

* When adding a new dependency to any `Project.toml` file, run `make distclean`, and then `make test/Manifest.toml`, `make docs/Manifest.toml`, etc. to recreate manifest files as necessary.

* In order to get the documentation (docstring) of any function, e.g., `some_func`, in any package in the test environment, e.g., `SomeLib`, run `julia --project=. -e 'using REPL; using SomeLib; print(Base.doc(SomeLib.some_func))'`

* Never commit any changes or ask to commit. I will always create git commits manually.
